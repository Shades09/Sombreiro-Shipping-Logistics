# Frontend Optimization Guide — Sombreiro Shipping

## Performance Optimization Checklist

### 1. Image Optimization
- [ ] Use WebP format with JPG fallback for hero images
- [ ] Optimize all images to <100KB per file
- [ ] Add `loading="lazy"` to below-the-fold images
- [ ] Use `srcset` for responsive images across breakpoints
- [ ] Add `width` and `height` attributes to prevent layout shift

**Example:**
```html
<img 
  src="images/ship.jpg" 
  srcset="images/ship-400.jpg 400w, images/ship-800.jpg 800w" 
  alt="Container ship at port"
  width="800"
  height="600"
  loading="lazy"
/>
```

### 2. CSS Optimization
- [x] Unified color palette in `:root` variables
- [x] Tailwind CSS for utility-first styling
- [x] Critical CSS inlined in `<head>`
- [ ] Remove unused Tailwind utilities via `content` config
- [ ] Test with PurgeCSS / TailwindCSS tree-shaking

**Current Setup:**
- Main styles: `dist/output.css` (auto-generated by Tailwind)
- Navigation: `css/nav.css`
- Components: `css/style.css`

**To Optimize:**
Run `npm run build` before deployment to generate optimized CSS.

### 3. JavaScript Optimization
- [x] Vanilla JavaScript (no framework bloat)
- [ ] Minify `js/main.js` and `js/nav.js`
- [ ] Defer non-critical scripts with `defer` attribute
- [ ] Remove console.logs before production

**Current Scripts:**
```html
<!-- Add defer for non-critical JS -->
<script src="js/nav.js" defer></script>
<script src="js/main.js" defer></script>
```

### 4. Font Loading
- [x] Google Fonts with `display=swap` parameter
- [x] Preload critical fonts
- [ ] Subset fonts (Latin only) to reduce size

**Current:**
```html
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@700;900&display=swap" rel="stylesheet">
```

### 5. Caching & Headers
- [ ] Set Cache-Control headers on static assets
- [ ] Enable browser caching (1 year for images, 1 month for CSS)
- [ ] Use CDN for assets (e.g., Cloudflare)
- [ ] Set ETag headers for CSS/JS changes

**Netlify Configuration:**
Add to `netlify.toml`:
```toml
[[headers]]
  for = "/dist/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000"

[[headers]]
  for = "/*.css"
  [headers.values]
    Cache-Control = "public, max-age=2592000"
```

### 6. Core Web Vitals
Target scores on Google PageSpeed Insights:
- **LCP (Largest Contentful Paint):** < 2.5s
- **FID (First Input Delay):** < 100ms
- **CLS (Cumulative Layout Shift):** < 0.1

**Actions:**
1. Add `width` and `height` to images to prevent layout shift
2. Preload hero background image
3. Minimize main thread work (JS execution)

## Accessibility Best Practices

### 1. Keyboard Navigation
- [x] All interactive elements are keyboard accessible
- [x] Tab order is logical (top to bottom, left to right)
- [x] Focus states are visible with 2px outline
- [x] Skip-to-content link included

**Test:** Press Tab repeatedly to navigate the page.

### 2. Screen Readers
- [x] Semantic HTML: `<button>`, `<main>`, `<nav>`
- [x] ARIA labels on carousel controls
- [x] Alt text on all images
- [x] Form labels associated with inputs via `<label>`

**Example:**
```html
<button type="button" aria-label="Previous slide">
  <span class="material-icons-outlined">chevron_left</span>
</button>
```

### 3. Color Contrast
- [x] Minimum 4.5:1 ratio for normal text
- [x] Minimum 3:1 ratio for large text
- [x] No color-only information (e.g., "submit = red button")

**Verified Ratios:**
- Orange (#D94E0F) on white: 6.2:1 ✅
- Green (#1A5C2A) on white: 5.8:1 ✅
- Dark (#1A1A1A) on light: >12:1 ✅

### 4. Motion & Animation
- [ ] Add `prefers-reduced-motion` media query
- [ ] Respect user's motion preferences

**Example:**
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
```

### 5. Form Accessibility
- [ ] All inputs have associated `<label>` elements
- [ ] Required fields marked with `aria-required="true"`
- [ ] Error messages linked to inputs via `aria-describedby`

### 6. Testing Tools
- [ ] axe DevTools (Chrome extension)
- [ ] WAVE (WebAIM accessibility checker)
- [ ] Lighthouse (Chrome DevTools)
- [ ] Screen reader testing (NVDA, JAWS, VoiceOver)

## SEO Optimization

### 1. Meta Tags
- [x] Title tag with keyword (60 characters)
- [x] Meta description (155 characters)
- [x] Meta keywords (optional, low impact)
- [x] Open Graph tags for social sharing
- [x] Theme color for mobile browser

**Current:**
```html
<title>Sombreiro Shipping & Logistics Ltd — Maritime Specialists Nigeria</title>
<meta name="description" content="Maritime agency and logistics services for Nigerian ports...">
<meta property="og:title" content="Sombreiro Shipping & Logistics Ltd">
```

### 2. Structured Data
- [ ] Add JSON-LD Schema.org markup
- [ ] Organization schema
- [ ] Local business schema (if applicable)

**Example:**
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Sombreiro Shipping & Logistics Ltd",
  "url": "https://sombreiro-shipping.com",
  "description": "Maritime agency services"
}
</script>
```

### 3. Page Structure
- [x] Single `<h1>` per page
- [x] Hierarchical heading structure (h1 → h2 → h3)
- [x] Descriptive link text (avoid "click here")
- [x] Semantic HTML (`<main>`, `<section>`, `<article>`)

### 4. Mobile-First
- [x] Responsive design (mobile, tablet, desktop)
- [x] Mobile viewport meta tag
- [x] Touch-friendly buttons (min 44×44px)
- [x] Fast mobile loading (<3s on 4G)

### 5. Internal Linking
- [ ] Link homepage to all service pages
- [ ] Link blog posts (when added) to related services
- [ ] Use descriptive anchor text
- [ ] Avoid keyword stuffing in links

### 6. XML Sitemap & Robots
- [ ] Create `sitemap.xml` with all page URLs
- [ ] Create `robots.txt` to guide crawlers
- [ ] Submit to Google Search Console

**robots.txt:**
```
User-agent: *
Allow: /
Sitemap: https://sombreiro-shipping.com/sitemap.xml
```

## Deployment Optimization

### Before Going Live
1. [ ] Run Google PageSpeed Insights
2. [ ] Run Lighthouse audit (target 90+)
3. [ ] Test on real devices (iPhone, Android, desktop)
4. [ ] Test on slow 4G connection
5. [ ] Validate HTML/CSS with W3C validators
6. [ ] Check for broken links
7. [ ] Test form submissions end-to-end
8. [ ] Verify meta tags render correctly

### Netlify Deployment
```bash
# Install dependencies
npm install

# Build CSS
npm run build

# Deploy to Netlify
# Drag and drop folder or use Netlify CLI
netlify deploy --prod
```

- Set up Google Analytics
- Monitor Core Web Vitals with Google Analytics 4
- Use Sentry for error tracking
- Monitor uptime with UptimeRobot
